HTTP Methods & Definitions

Header Field Definitions

  • Accept
    • it is used to specify certain files that can be accepted for the response.
    • the q parameter, this is used for indicating the priority of the files that the server should return. It's value ranges from zero to one (0-1), with 1 being the default value. The higher the q value the higher the priority.
    • if there is no Accept header field, then it is assumed that the user accepts all file types.
  • Accept-Encoding
    • it is similar to the Accept header, but limits the content-encodings of the files.
  • Accept-Ranges
    • allows the server to specify its range request's acceptance for the file.
  • Authorization
    • contains the credentials for the authentication information of the user in the desired resource that is requested.
    • if the request is authenticated and the resource is specified, the same username and password should work for any other requests inside that resource.

Method Definitions

  • Safe Methods
    • is a read-only file. It does not take any othre action other than retrieval.
    • this only uses the GET, HEAD, OPTIONS &, TRACE methods.
  • Idempotent Methods
    • even if the methods get repeated, it will always result to the same output.
    • this method uses the GET, HEAD, OPTIONS , PUT, DELETE &, TRACE methods.
  • Cacheable Methods
    • storage that is accessible to the client to be stored for future use.
    • this method can also save up bandwidth.
    • this method only uses the GET, HEAD &, POST methods.
  • Standard Methods
    • GET - must be supported by every server.

      - fetch requirement to all servers.

    • HEAD - must be supported by every server.

      - same as the GET method but doesn't have a body.

      - is used to retrieve meta data.

    • POST - is used to make changes in the target resource.

      - enclosed as the message body.

      - performs resource and specific processing.

      - enclosed as body.

    • PUT - generates a resource

      - this method is not supported by default and is difficult to implement

    • DELETE - removes a resource

      - this method is not supported by default and is difficult to implement

    • OPTIONS - tells what you can do to the resource.
    • TRACE - allows the server to echo the request.
    • CONNECT - create a secure connection and establish a tunnel.
