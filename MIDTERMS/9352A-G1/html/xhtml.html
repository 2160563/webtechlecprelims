XHTML & HTML

Difference of XHTML & HTML

XHTML

  1. Maintenance is easier as well as learning it and teaching it
  2. Case sensitive (lower-case)
  3. Requires strict and proper coding practices
  4. Best for computer screens

HTML

  1. Compatible with all types of browsers
  2. Case insensitive
  3. Provides the users with an ease of integrating different media
  4. Works better on various screens (mobile phones & tablets)
XHTML's Most Important differences from HTML
  • Document Structure
    • XHTML DOCTYPE is mandatory 
      <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"
"http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
    • The xmlns attribute in <html> is mandatory 
      <html xmlns="http://www.w3.org/1999/xhtml">
    • <html>, <head>, <title>, and <body> are mandatory 
          <head> 
        <title>Title of document</title> 
    </head> 
    <body>
        some content 
    </body>
</html>
  • XHTML Elements
    • Elements must be properly nested
      Invalid: 
      <h2><em>Header1</h2></em>
      Valid: 
      <h2><em>Header1</em></h2>
    • Elements must always be closed
      Invalid: 
      <em>Text
<br>
<hr>
<img src="" alt="">
      Valid: 
      <em>Text</em>
<br />
<hr />
<img src="" alt="" />
    • Elements must be in lowercase
      Invalid: 
      <B>Hello</B>
      Valid: 
      <b>Hello</b>
    • Documents must have one root element
  • XHTML Attributes
    • Attribute names must be in lower case
      Invalid: 
      <div WIDTH="50%">
      Valid: 
      <div width="50%">
    • Attribute values must be quoted
      Invalid: 
      <div width=50%>
      Valid: 
      <div width="50%">
    • Attribute minimization is forbidden
      Invalid: 
      <input type="checkbox" name="vehicle" value="car" checked />
      Valid: 
      <input type="checkbox" name="vehicle" value="car" checked="checked" />
