HTTP Methods

Method Definitions

  • Safe Methods
    • is a read-only file. It does not take any othre action other than retrieval.
    • this only uses the GET, HEAD, OPTIONS &, TRACE methods.
  • Idempotent Methods
    • even if the methods get repeated, it will always result to the same output.
    • this method uses the GET, HEAD, OPTIONS , PUT, DELETE &, TRACE methods.
  • Cacheable Methods
    • storage that is accessible to the client to be stored for future use.
    • this method can also save up bandwidth.
    • this method only uses the GET, HEAD &, POST methods.
  • Standard Methods
    • GET - must be supported by every server. It retrieves whatever information is identified by the Request-URI.
    • HEAD - must be supported by every server. It is identical to the GET method except that the server must not return a message-body in response. Also, it is used to retrieve meta data.
    • POST - is used to make changes in the target resource. It performs resource and specific processing. Enclosed as the message body.
    • PUT - generates a resource

      * this method is not supported by default and is difficult to implement

    • DELETE - removes a resource

      * this method is not supported by default and is difficult to implement

    • OPTIONS - tells what you can do to the resource.
    • TRACE - allows the server to echo the request.
    • CONNECT - create a secure connection and establish a tunnel.
