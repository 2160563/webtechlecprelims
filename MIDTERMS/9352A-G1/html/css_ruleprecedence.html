CSS Rule Precedence

What is Rule Precedence?

An HTML element may be the subject of the selectors of multiple style rules. When such rules target different properties, their effects cascades, and when the styles involve the same property, they conflict, and must be resolved such that only one style is applied.

Resolution:

  • by Origin and Importance

    • The origin of a declaration is based on where it comes from and its importance is whenever it is declared important. The precedence of the various origin is in descending order:

    1. Transition Declaration
    2. Important user agent declarations
    3. Important user declarations
    4. Import override declarations
    5. Import author declaration
    6. Animation declaration
    7. Normal override declaration
    8. Normal author declaration
    9. Normal author declaration
    10. Normal user agent declaration

    Declarations from origins earlier in this list win over declarations from later origins.

  • by Specificity (Calculating a selector’s specificity)
    1. count the number of ID selectors in the selector (=a)
    2. count the number of class selectors, attribute selector, pseudo-classes(=b)
    3. count the number of type selectors and pseudo-elements in the selector(=c)
    4. ignore the universal selector
  • by Order

    • If an element declared twice or more with the same property but different value, the last declaration will be followed.

    p {
    color: red;
    color: blue;
}

    In the example above, the color of the content of the paragraph will be color blue.
