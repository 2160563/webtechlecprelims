Hypertext Transfer Protocol (HTTP)

What is HTTP?

  • It is based on a client-server architecture.
    • Servers - the origin server.
    • Clients - also known as User Agents.

      - these are the web browsers and the web crawlers.

  • Communicates usually through TCP/IP.
  • Communicated through hosts and clients via request and response.
  • Uses port 80 as it's default.
  • A communication protocol used to access resources on the World Wide Web . Invented by Sir Tim Berners-Lee at CERN in 1989.

HTTP Version History

  • HTTP 0.9(1991)
  • HTTP 1.0(RFC 1945, May 1996) - get, header, post
  • HTTP 1.1(RFC 2068 Jan 1997; RFC 2616 June 1999), RFC 7230 - 7235(June 2014)
  • HTTP 2(RFC 7540 May 2015) - additional mechanism in the transmission of data with header compression that allows multi pleasing which is more enhanced than the previous versions.

HTTP is a stateless communications protocol and uses request messages.

Its functionalities include:
  • Cache Control
  • Content Media type (MIME)
  • Persistent Connection, thus it can be reused
  • Request Pipelining/Multiplexing - can have many request at a time.
  • Authentication/Authorization - talks about the security and identifies if the client can be verified and authorized at the same time.

HTTP Resource Addressing

  • Scheme (http or https)
  • Authority - domain name (resolved to an IP address DNS), host, post number and user information or authentication credentials.
  • Path to resource (to resolve relative to the document on the server). This may be a static or dynamic resource.
  • Query - set of key = value pairs use:
    • Ampersand (&)
    • Delimiter (?)
    • URL-encoded (>)
  • Fragment identifier - subsection in a document, hash sign (#) symbol.

Request Message

  • The request message from the client to the server
  • First line: Request Line
    • Syntax: METHOD_TARGET_PROTOCOL
    • Note: Please keep in mind that the underscores (_) are considered as SP or spaces to separate the codes from one another.

Response Message

  • After the request message has been sent and interpreted, the server will now respond with an HTTP response message
  • First line: Response Line
    • Syntax: PROTOCOL_STATUSCODE_TEXTUALPHRASE
    • Note: Please keep in mind that the underscores (_) are considered as SP or spaces to separate the codes from one another.

Message Headers

  • Syntax: FIELD-NAME ":" FIELD-VALUE
Three (3) types:
  • General - related to the characteristics itself.
  • Entity - characteristics of the body
  • Response
